Day 2: Python Data Structures, Loops & Functions

1. Python for Loops & Formatting

The range() function: Instead of manually writing out the start, condition, and increment like in other languages, Python uses range() to easily loop through a sequence of numbers.

Print Formatting: You learned how to display output cleanly using commas to separate values, as well as f-strings for dynamic, formatted text.

2. Derived Data Types (The Core Structures)

Lists []: Created with square brackets. They can hold different types of data at once, they maintain their order, and they are mutable (you can add, remove, or change items later). They are accessed using an index starting at 0.

Tuples (): Created with parentheses. They are ordered like lists, but they are immutable (they cannot be changed once created). Because they are locked, they are faster and use less memory than lists.

Sets {}: Created with curly braces. They are completely unordered, cannot be accessed using an index, and most importantly, they automatically remove any duplicate values.

Frozen Sets: A variation of sets that acts like a tuple—it is unordered and removes duplicates, but it is immutable and cannot be changed after creation.

3. Dictionaries

Key-Value Pairs: Dictionaries store data by linking a specific "key" to a "value" (like pairing the key "Student_Name" with the value "Arnav").

Data Management: You practiced creating lists containing multiple dictionaries to manage dynamic runtime data, such as calculating total marks and averages for different students.

4. Functions & Variable Scope

Functions: Reusable blocks of code that take in specific inputs (parameters) and send back a calculated result (return values).

Variable Scope: You encountered real-world debugging by dealing with local variables. Variables created inside a function only exist inside that function, and trying to access them from the outside will cause scope errors.
